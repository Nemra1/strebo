Use-Case Specification: Show trends of supported networks

 

1.            Show trends of supported networks

1.1          Brief Description

Show trends of supported networks is an important Use-Case, because it serve one of the main functions of strebo.
This Use-Case is the main function of the trend board and it gives the user the possibility to see the trends of all the aviable social networks by using the different specific APIs.

2.            Flow of Events

2.1          Basic Flow

The use case starts with the user interacting with the website by opening the trend board by either pressing the button or by opening strebo in his browser. A mockup of the said button as well as of the website containing the trend board can be found in figure 2.1.1.
This interaction is the trigger for the system to connect to the single supported networks by using the specific APIs. This procedure does not need any credentials of the user.
After the successful connection process, the system automatically searches for the trending contents based on the single APIs and networks. The found posts, tweets etc. will then be embedded in to the trend board and so be made available for the user to look at.
The graphical description of this use case can be found in figure 2.1.2.

3.            Special Requirements

3.1          Internet access

Without access to the internet it is not possible for the system to interact with the social networks. So there is now way to get the trending content.

3.2          Availability of the social networks

If one or more of the social networks are temporarily not available, the content of these networks can not be embedded in the trend board.

3.3          Functions of the APIs

The single APIs of the different social networks have to enable the possibility of searching for trending content of posts, tweets etc.

 

4.            Preconditions

There are no preconditions for this use case. It can be performed at any state of the application.

5.            Postconditions

5.1          Showing the trend board

After this use case the system is showing the user the trend board with the found trending posts of the single social networks.

5.2          Showing failure message

If an error occurred within the process of the use case, depending of the occurrence error the social network or the server shows the user a message with information about the occurred error.
The process than is interrupted and has to be started again.
