Use-Case Specification: Connect to Supported Social Media Network Accounts

 

1.            Connect to supported Social Media Network Accounts

1.1          Brief Description

To make a personal dashboard with personal relevant content for the user available it is needed to connect its social media account from supported social media networks to strebo.

2.            Flow of Events

2.1          Basic Flow

The use case starts with a click on a social media network in the social media bar in the bottom of the page. If the user is already connected to the network it happens nothing. If not it popups a login page of the corresponding network. Maybe the user have to input his login credentials and accept that strebo get access to personal information of this account. If the login procedure was successful the personal dashboard will be available and the user can see that he is connected to this special social media network in the 'connector bar'. .

2.2          Alternative Flows

No Alternative Flows determined

3.            Special Requirements

3.1          Internet access

Without access to the internet it is not possible for the system to interact with the social networks. So there is no way to login to a social media network.

3.2          Avaialability of the social networks

If a social network is temporarily not available, the user can't connect to it.

3.2          Social Media Account

The user only can connect to a social media network if he has a account in this special network. In addition he has to give strebo the access rights to several personal information such as his personal feed.

3.3          Functions of the APIs

The single APIs of the different social networks have to enable the possibility of SSO connections to applications.

 

4.            Preconditions

There are no preconditions for this use case. It can be performed at any state of the application.

5.            Postconditions

5.1          Showing the trend board

After a successful flow of this use case, the system is able to add the social network to the personal dashboard.

5.2          Showing failure message

If an error occurred within the process of the use case, depending of the occuring error the social network or the server shows the user a message with information about the occured error.
The process than is interrupted and has to be started again.
